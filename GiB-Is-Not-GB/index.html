关于 GiB 与 GB 的区别

发表于 2016-08-01  |  2017-01-10

之所以想到这个话题，是因为好几次有人问我：为什么用 macOS 时，经常会遇到文件下载完成后变大的情况？其实，并不是文件变大了，而是单位变了。

想必，即使是对计算机不熟悉的人，应该也知道硬盘容量和操作系统显示容量是不一样的，其中前者是 1000 进位，后者是 1024 进位。但至于为什么不一样，这两套单位有什么意义，可能就一知半解了。甚至有人认为，这是硬盘商的阴谋，故意迷惑消费者。可如果我说：硬盘商从始至终都是对的，错的是微软，你们会有什么想法？

这可不是我故意乱搞大新闻，而是事实。实际上，GB 就是 1000 进位的（准确说是十进制），1GB = 1000MB，1MB = 1000KB，1KB = 1000B。因此，硬盘商并没有故弄玄虚，而是实事求是地标出了硬盘容量——1TB，真的就是 1TB，1TB 真的就只是 1000GB，依此类推。

那为什么操作系统里，1GB = 1024MB 呢？实际上，这里所说的操作系统，只有 Windows 一家。换言之，是微软一手造成了这个误解。

计算机世界确实还有一个 1024 进位的单位体系（准确说是二进制），但它不是 GB，而是 GiB，也就是 Giga binary Byte 的缩写，比 GB (GigaByte) 中间多了一个 binary（二进制）。而 1GiB = 1024MiB，1MiB = 1024KiB。这才是 1024 的世界，比 1000 的那个家伙多了一个 i 字母。可微软却没有正确标识，他们把 GiB, MiB, KiB 全都标成了 GB, MB, KB。因此，在 Windows 系统中显示的 1GB，实际上都是指 1GiB，和硬盘的 1GB 并不是一个单位，于是误解就产生了。

微软这么做的用意大概是为了让格式更加简略。也许他们认为计算机的世界本就该是二进制，并不需要多加一个 i 来强调，于是干脆鸠占鹊巢，把 GB/MB/KB 这套单位符号给占用了，并希望借此消灭这套十进制的单位。但可惜，它并没有被消灭，反而十分广泛地被硬盘厂商、其他操作系统使用着，也正因此，由于两个单位都活跃在我们的计算机世界，却因为 Windows 的普及经常使用同一套单位符号，混淆和误解也就随之产生了。

既然两套单位同时存在的事实已成定局，那么我们只要正确标注、不张冠李戴，就可以避免出现问题。例如，苹果在 macOS 里就和微软一样，用的也是 GB/MB/KB 这套单位符号，但 GB 就是真正的 GB，和硬盘商所使用的单位完全一致。所以 1TB 的硬盘，在 macOS 上就真的显示为 1TB（实际会略小，有一小部分会留用作保护区）。

这就好比距离的单位，有公制，也有英制。对于同一个长度，你可以用公制标识，也可以用英制标识，这都无伤大雅，只需正确使用即可。微软的做法，就好比明明使用的是英制，却用了公制单位，把 30.48cm 标注为 1 米（本应为 1 英尺）。明明是 1GiB，却写成 1GB，误导了消费者。

总结

文件大小有两套单位系统，一套是十进制，一套是二进制，它们的符号有所不同。十进制 1GB = 1000MB，1MB = 1000KB；二进制的中间有 i，1GiB = 1024MiB，1MiB = 1024KiB。微软用的是二进制那套单位，符号却没写 i（变成了十进制的单位），并随着 Windows 的普及，把这个问题也「普及化」了。

回到开头那个问题：为什么用 macOS 时，经常遇到文件下载完成后变大的情况？因为 macOS 使用真正的 GB/MB/KB，也就是 1000 进位。当你下载了一个 1GB 的文件，它其实是 1GiB 大小，所以在 Finder 上就被换算成 1.07GB 了。
