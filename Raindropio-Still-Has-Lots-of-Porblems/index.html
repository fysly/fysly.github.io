Raindrop.io 的新版设计仍然值得吐槽

发表于 2016-10-02  |  2017-01-10


是的，这个界面很简洁、好看——可是「下一步」按钮呢？没了，原本有的，现在却没了。现在唯一的保存方法是按回车键。但是，实际上在图中这一状态下，你即使按遍所有按键，也无法保存。因为，「按回车键进入下一步保存」只在 link 输入框生效，一旦把光标定位到 tags 输入框，就无法保存了。而我习惯在输入链接地址后顺手添加 tag，被这一糟糕设计所害，导致我摸索了很久也无法找到保存的方法，直到反复测试时随手打了一个链接，没有添加 tag 而直接按下回车……终于柳暗花明了。

不得不说，这种体验实在是太糟糕了。

但这还没完。你能想象吗？像「清空垃圾箱」这种后果严重、且无法恢复的操作，Raindrop.io 居然敢不放「确认对话框」！是的，没有「确认对话框」，也就是说，当你点下、可能只是误按了「清空」，于是就真的空了，再也无法恢复。

怎么讲？Raindrop.io 还是老样子，理念很现代、功能很全面，设计也很好看，但总是在实际体验中狂打你的脸，让你对之前给予它的高评价感到羞愧。

背景补充

我不知道这款名为 Raindrop.io 的书签收藏服务到底有多少用户量，毕竟在这个小众的领域，过去更出名的还是 Pinboard 和 Delicious。之所以说小众，是因为书签收藏这个需求其实并不是特别强烈，尤其是现在浏览器自带的收藏夹就已经足够好，而且支持多平台同步的情况下。如果想要阅读、收藏文章，则是 Pocket 或 Instapaper 之类的「稍后再读」服务更加合适；如果想要保存文本并做上批注，则更适合用 Evernote 等笔记服务。相比之下，单纯只保存网页似乎确实显得有些小儿科，而且网页本身就是不确定的，随时可能被站方删除、修改，导致原本保存的内容失效。

但我还是需要一款书签收藏服务。确实，那些经常打开的，我会放在浏览器收藏夹里；那些宝贵的、唯恐丢失的，我会摘录存在笔记里。但除了这二者以外，还有很多网页值得收藏。举个最简单的例子，那就是知乎的回答。虽然知乎本身就有收藏夹功能，但做得实在太简陋了，每次在里面找某个回答都非常麻烦。因此，我需要一个支持 tag 和搜索功能的收藏夹。

同理，并非每个网站都像知乎一样具有收藏功能，就算有，也不一定做得完美。那么，如果用一款专门的工具做收藏，把这些杂七杂八的网站里的好内容全部收藏到一起，然后用打上 tag，并写上简单的描述，随时可以用关键词搜索找到对应内容，那就能解决这一痛点了。

最初知道 Raindrop.io 是通过少数派的这篇文章，发现它能完美满足上述这些需求，且多平台支持，随时可以找到需要的内容，还具备文本显示优化的功能，可以取代「稍后再读」服务，再加上出色的颜值，简直完美。于是，我马上选择了它。可以说，它是 Delicious 的加强版（由于支付问题，我并没有用过更加低调但口碑极好的 Pinboard，但从其简陋的官网来看，它显然没有像 Raindrop.io 那样全面和支持多平台）。

但在使用过程中，我很快就发现了 Raindrop.io 的一些问题。首先是笨重。虽然好看，但这套 UI 界面显然颇占系统开销，有时会出现卡顿现象，但更多、更麻烦的，则是崩溃（主要是浏览器扩展）。于是我放弃了使用 Chrome 扩展，改用客户端，总算是一定程度了解决了问题。但总的来看，Raindrop.io 仍然显得笨重、缓慢，以至于虽然我没有抛弃它，但使用频率十分低，总是习惯性地使用浏览器自带收藏夹。

最后，终于等到开发商发了邮件，声称他们对 UI 界面进行了重制。从效果图上来看，确实轻了不少，但没有想到，实际使用中会遇到这样令人难堪的问题。
