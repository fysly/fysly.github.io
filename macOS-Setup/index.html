macOS 必要设置指南

发表于 2016-01-31  |  2017-01-14

不是上手指南，也不是小技巧收集，仅根据个人经验，对原生的 macOS 进行必要的设置，以提升使用体验。

由于 macOS 每年都会有一次大版本更新，所以本文内容随时可能失效，我会根据情况有选择地更新补充。

注：$ 符号代表后面跟着的是命令行，你可以直接复制粘贴该行命令[1]到终端，然后按回车键执行。

系统偏好设置

通用

如果是 Retina 屏幕的 Mac，可以考虑取消「使用 LCD 平滑字体」的勾选。

LCD 渲染，更准确的说法是亚像素 (subpixel) 渲染[2]，会使字体变得更粗、且边缘出现彩色。它的好处是使字体更加清晰，对低分辨率屏幕来说十分有用，但 Retina 屏幕就没有这个必要了，除非你对这种显示效果有特殊偏好。

可选：按照这篇文章的提示，删除「华文黑体」等已经过时、但由于被某些网页使用，仍然会强制出现的字体。

Dock

Dock 可根据个人喜欢任意添置应用，但建议只选择最常用的，诸如 Launchpad 等可以由快捷键直接打开的界面，不必为其浪费 Dock 空间；macOS Sierra 新增了 Siri 功能，由于可通过右上角的状态栏触发，因此也可以把 Dock 上的图标删除，为更加重要的应用腾出空间，减少视觉干扰。

此外，可以在系统偏好设置中勾选「自动显示和隐藏 Dock」。当光标移动到屏幕下方时，自动显示 Dock，离开便自动隐藏，增大屏幕面积利用率。

Mission Control

将 Dashboard 关闭，它已经和新一代 macOS 毫无关系了，即将成为历史。

语言与地区

列表排列顺序建议改为「通用」，而非「中文（拼音顺序）」等其他排序方式。

Spotlight

Spotlight 非常强大，不仅找文件非常方便、可以快速打开应用程序，还可以用来查词典、查汇率，当计算器……

如果你觉得 Spotlight 查找文件呈现的结果非常多，可以在这里设置去除不想要的部分。

通知

很多应用都应该允许通知，但只有少数需要将通知内容保留在通知中心。对于时效性非常强的通知，可以将该应用取消勾选「在通知中心显示」。例如压缩工具，就只需要一条横幅通知我压缩已经完成即可，并不需要将这条信息一直保留在通知中心（过后再看也毫无意义，对吧？）。

这样做可以使通知中心保持干净、有序，只保留邮件等更为重要的通知，更清楚地查看。

快捷键

快捷键在系统偏好设置——键盘里。首先请看苹果的官方说明文件，以免看不懂下文 ⌥ 等按键符号。

  • 全键盘控制
    在系统偏好设置——键盘——快捷键中，把全键盘控制改为「所有控制」。对于深蓝色背景的按钮（默认），可以按回车键进行快捷点击；开启全键盘控制后，可以通过 TAB 键切换按钮，使其出现浅蓝色边框，此时可用空格键快捷点击。

  • 输入源（输入法）
    输入法有两种切换，一种叫「选择上一个输入源」，一种叫「选择『输入』菜单的下一个输入源」。默认只开启了第一种，快捷键为 ⌘+Space，你可以按住 ⌘ 键，再多次点击空格键从弹出的输入法选择框中选择。但我把这一种关了，用第二种取而代之。因为这种切换是「智能」的，智能往往也意味着「自作多情」、「自作主张」。

    第二种就非常死板，按顺序切换，如果目前是第一个输入法，按一次输入法就切换到第二个，再按一次就切换到第三个，再按就切换回第一个，永远也不会搞错。如果你和我一样，选择关闭第一种，开启第二种，不妨把前者的快捷键换给它（看下文）。

  • Spotlight
    对于没有添加中文输入法的 macOS，Spotlight 默认快捷键是 ⌘+Space，比如英语国家就都是这样。建议中文用户也按照英文系统下的快捷键设置，将 Spotlight 快捷键改为 ⌘+Space，然后将切换输入法改为 ⌥+Space。

  • 复制、粘贴、剪切（移动）
    macOS 复制的快捷键是 ⌘+C，粘贴是 ⌘+V，二者和 Windows 没什么区别。但剪切则不同。

    其中，文本的剪切和 Windows 相同，快捷键为 ⌘+X；文件则不能剪切，只能移动。Windows 把「移动文件」也命名为类似文本的剪切、粘贴，而 macOS 等系统则没有这样做，移动就是移动。

    这就是为什么在 Finder 里「剪切」是灰色的，无法使用。首先需要解释一下，菜单栏是所有软件公用的，所以编辑里会有剪切（其他软件会用到它），但由于文件并没有剪切这个概念，所以当切换到 Finder 时它就会变成灰色，处于不可用状态。

    总而言之，如果想对文件进行 Windows 式的剪切、再粘贴，请直接无视这个「剪切」（它只用于文本编辑），而是使用「移动」。移动文件的方法为：先复制（而不是剪切），然后再移动（而不是粘贴）。移动的快捷键是⌥+⌘+V，就是在粘贴的基础上多按一个 Option 键。

文本

文本也在系统偏好设置——键盘中。

建议关闭「自动拼写纠正」、「使用智能引号和破折号」。例如后者会使某些应用在输入时强行转换引号，导致代码出现错误。

触摸板

macOS 现在默认按下触摸板才是点按，建议添加过去一直使用的「轻点」，更加省力，方法为勾选「轻点来点按」。

同时不知为何，苹果把「三指拖动」这么重要的功能也默认关闭了，强烈建议开启。在系统偏好设置——辅助功能——鼠标与触控板——触控板选项中勾选「启用拖动」。

用户与群组

可以修改「登录项」（即开机自动启动的程序），将不需要开启自动启动应用去除（按下方的 - 号），或者添加希望开机自动启动的程序（按下方的 + 号）。

终端 (Terminal)

GUI 与 CLI

macOS 的强大之处在于，一方面由于苹果的努力，使其拥有了一个好看且好用的 GUI 桌面环境，以及一大票精致的桌面应用；另一方面，它继承了 UNIX 精髓，承接其相关的软件、开发生态，并和同为类 UNIX 的 Linux 兼容性良好，方便移植。如果对后者视而不见，相当于浪费了 macOS 的一半。

前文所有设置都在「系统偏好设置」这个 GUI 程序中。但操作系统如此庞大，显然包含所有参数、设置。这就是 GUI 的局限：设置那么多，总不可能每个都写到 GUI 里面，做成选项或按钮吧？哪怕是 Windows 那样把 GUI 发挥到极致的系统，也仍然有很多操作必须通过命令行完成（例如刷新 DNS，打开笔记本的无线热点，虽然近年来已经有很多第三方公司推出了 GUI 软件来实现这一功能）。

Terminal 与 Shell

那么如何使用命令行呢？简单地说，UNIX 或类 UNIX 系统都有一种叫做 shell 的程序，我们可以在 shell 中使用命令行来和操作系统的核心沟通。由于我们现在的操作系统都不再是命令行界面，而是一个 GUI 桌面，所以为了使用命令行程序 shell，我们还需要一款叫做终端的程序（可以看作是一个模拟器）。终端是一个 GUI 程序，所以长得也和你平时用的程序差不多，就是一个窗口，可以用鼠标和键盘对其进行控制，而窗口里面显示的，就是 shell 了。

终端有很多选择，建议直接使用 macOS 自带的那个同名程序（对，就叫终端.app ）即可，已经足够好用，程序员则可以选择 iTerm2。目前 macOS 默认的 shell 名叫 bash，不过不失。我们可以切换成更强大的 zsh，系统中已经自带。

可以打开 Spotlight（前面刚说了快捷键），输入 ter 三个字母，然后回车即可快速打开终端。建议在终端的偏好设置中，把默认的描述文件改为 Pro，然后再到描述文件中，把 Pro 的字体字号改大些，窗口大小也改大些。

zsh 与 oh-my-zsh

zsh 也许是最好的 shell，连名字都透着一股终极的味道[4]。但它需要配置、折腾，上手也并不容易。好在这些麻烦事有位大神已经替我们包揽了其中绝大部分，于是就有了 oh-my-zsh 项目。只要安装了它，我们就可以直接使用已经配置好（也可以进一步修改）的 zsh 了。

你可以在oh-my-zsh 的 GitHub 页面查看详情，根据提示安装。安装前请确认已拥有 Xcode Command Line Tools，然后根据输出的提示，输入密码或按键确认。

有了 zsh，使用命令行将更加轻松愉快。例如输入 cd dow，然后按下 Tab 键，你会发现 shell 自动帮你补全了目录名 Downloads/，虽然 bash 也有这样的功能，但 zsh 更加强大，而且可以智能识别大小写。

自动补全是非常重要的功能，尤其是对新手来说，可以省很多事，还可以防止输错，所以凡是输文件、目录名，都可以输前几个字母然后按 Tab 自动补全。

包管理器 Homebrew

包管理器

包管理器，或者叫软件包管理工具，你可以把它理解为类似 App Store 的一种软件，可以在里面下载、安装、管理各种软件。区别在于，App Store 里面都是 GUI 应用程序，而这里所说的包管理器，管理的则是 CLI 程序。对于 macOS 来说，包管理器主要就两个选择，推荐使用 Homebrew。

为什么要用包管理器？其实道理就和苹果为什么要推出 Mac App Store 一样，这样用户就有了一个专门的地方统一安装应用程序，不再需要每安装一个程序，都去谷歌搜索、找到官网、进去查看安装方法、最后手动下载安装。例如刚才的 oh-my-zsh，我们也是先找到官方网站，然后再从官网处获取安装方法，进行安装。如果所有软件都这种操作，显然非常麻烦。

aria2

Aria2 是一款 CLI 界面的下载工具，但它提供了一个 RPC 模式，从而可以使用其他开发者做好的网页 GUI 界面来进行操作。不过个人觉得命令行反而更加简单，并不需要打开网页去进行管理。

Aria2 也有 Homebrew 版本，可以直接用命令 brew install aria2 来安装它。它使用配置文件进行设置。关于配置，可以在网络上查找，也可以直接使用我的配置。如果不会下载，也可以在本地创建文件，然后把文本复制、粘贴进去，保存。

配置文件必须放在 /Users/你的用户名/.aria2/ 目录下才能生效。注意，以 . 开头的都是隐藏文件夹，在 Finder 内默认状态下是看不见的，建议通过命令行来操作：

  • 打开终端，输入 mkdir .aria2 并回车
  • 输入 mv 并空格，然后把 aria2.conf 从 Finder 拖拽到终端，松开手指，系统会自动把文件路径输入
  • 最后输入 .aria2 才回车

这条命令就是把 aria2.conf 文件 mv 到 /.aria2 文件夹去，以后启动 aria2 就会自动使用这个配置了。

Aria2 的使用非常简单，对于普通任务来说，你只需要使用例如 aria2c http://yigelizi.com/yigelizi.txt（这个地址当然是假的啦）这样的命令即可下载文件。

如果是百度网盘，可以使用 acgotaku 开发的浏览器插件，获得文件的正确下载地址，从而使用 aria2 下载。关于 aria2，你还可以查看这篇文章来了解更多。文章中前面几步是安装 Homebrew，你已经安装了，可以直接跳过。

mpv 是一个开源的视频播放器，非常好用，强烈推荐。它有 Homebrew 版本，可以直接通过 Homebrew 安装。当然，正如前文所说，Homebrew 是管理 CLI 程序的，而我们看视频总不能每次都打开终端对吧？所以 mpv 很人性化地提供了一个打包成 GUI 程序的功能。当我们使用命令然后时，后面要附加参数 --with-bundle，这样 Homebrew 在安装 mpv 时就会将其和其所依赖的程序全部打包成一个应用程序，也就和普通程序一样了，可以直接双击一个视频打开播放。所以完整的安装命令为：

值得注意的是，mpv 依赖很多程序。「依赖」是开源软件的一种风格，举例来说，某软件 A 提供了一个非常好用的功能，于是很多其他软件如果需要这个功能，那么直接调用 A 就行了，不需要自己再重新写代码。这样做的好处就是节省了人力（编程时间）物力（磁盘空间）。而 mpv 也是如此，作为视频播放器，其实它只负责播放这一核心功能，而其他子功能则都是直接调用其他开源项目，例如 xxx 是一个字体渲染程序（和字幕显示有关），yyy 是一个开源的 PNG 编码工具，mpv 用来截图，并以 PNG 格式保存。zzz 则是……依此类推。之所以说节省了磁盘空间，是因为如果之后再安装某个软件，也用到了 xxx 这个软件，由于你已经有了，就不需要再安装。也就是说，一份就可以被多个软件使用，而不需要每个软件都自己准备一份。

但问题在于国内访问这些外国网站下载文件速度极不稳定，而 mpv 依赖的软件实在是太多太多太多了，以至于安装过程会变得非常漫长，随时可能网络不顺畅而降到零速度。我们可以随时用命令 ⌃+C 来中止命令，然后按方向上键重新选择之前的命令（按一次是上次的命令，按两次就是上上次的命令，依此类推）然后回车，继续安装。我在帮某人的电脑安装 mpv 时，由于网络及其恶劣，反复中断、继续十多次，才得以成功安装。

安装完成后，由于已经软件打包，所以可以用命令 brew linkapps mpv 将 mpv 软链接到应用程序文件夹下，之后就可以将其设置成默认程序了。

mpv 同样使用配置文件进行配置。这里同样给出我的配置，具体使用方法同 aira2 的配置文件。在本地创建一个 mpv 文件夹，然后在里面创建这两个文件。这次配置文件需要放到 /Users/你的用户名/.config/mpv 下。步骤如下：

  • 确认在个人用户目录（不确定的话直接输入 cd 并回车即可）
  • 输入 mkdir .config 并回车
  • 输入 mv 并空格，然后在 Finder 把 mpv 文件夹拖拽到终端，获取文件夹路径
  • 最后输入 .config 并回车

默认程序

我们可以选中一个文件，右键，单击「显示简介」，然后更改打开方式，并点击「全部更改」。这样相当于设置了此类文件的默认打开程序。

依此类推，其他后缀的文件都可以这样修改。另外，默认浏览器的设置在「系统偏好设置」的「通用」里；默认邮箱的设置在邮件 (Mail.app) 里。

其他必备软件

  • Chrome
    拓展（插件）决定一切。

  • F.lux
    色温调节工具，可以在夜间减少蓝光，使屏幕变成暖黄色，保护视力。既然 iOS 9 也增加了这个功能，可能 macOS 也不远了。

  • Rime
    Rime 输入法，Mac 上叫鼠须管，Windows 上叫小狼毫，Linux 上叫中州韵。它的非常强大之处在于支持多种输入方式，包括拼音、双拼、五笔、笔画……它的缺点是没有 GUI 设置界面，所以所有设置都是通过配置文件生效（又是配置文件，现在已经不陌生了吧），设置完便可以尽情享受了。
    关于 Rime 可以查看我这篇文章。

  • Sublime Text 3
    强大的文本编辑器，好看又好用，又不像 Atom 那样占资源、缓慢。上面提到的那些配置文件都可以用它来修改。

  • Typora
    一款 Markdown 编辑器。Mac 作为诸多作家、程序员的首选，编辑器可谓数不胜数，为什么选择它？首选，它是免费的，其次，它的所见即所得在 Markdown 中非常少见，使得新手也不再有陌生感，十分容易上手。

  • 1Password
    密码管理工具。

  • Bilibili for Mac
    第三方 B 站客户端，相当于一个浏览器（Safari 内核），并伪装成了官方手机客户端，从而可以使用 HTML5 播放视频，告别 Flash，告别发热。里面的播放器其实就是 mpv，还支持 you-get 等模块获取网页中的视频地址（所以并非只支持 B 站使用）。下载视频则是用 aria2 来实现（又是一个熟悉的名字）。

    可选：使用 Macs Fan Control 调节风扇转速，更全面地告别发热。

  • Leanify
    文件（主要是图片）减小体积、压缩工具，整合了诸多开源项目。虽然是 CLI 程序，但功能强大且支持批量处理（可以用 *.png 等通配符或直接将文件夹拖入终端）。

  • Dropbox
    世界上最好的云同步工具。既然苹果不收购它，那我们必须继续用它，哪怕被墙，也是如此。

  • 科学上网工具
    没什么可说的。

    可选：使用 @chengr28 的 DNS 防污染工具，获取正确的 DNS 解析结果。
