Tab Bar 就是好设计

发表于 2017-01-19  |  2017-01-19

最近这个问题下，我看到了这么一些回答。

这次更新真的是晴天霹雳。
首先，个人非常讨厌底部tab的风格，很难用，而且极其丑。要知道当初不用有道wiz很大一部分原因就是因为那个模式我觉得一点都不效率。 🔗

搜索单开一个底部tab（黑人问号？）作为笔记，不是发文章的媒体，下拉刷新没那么重要，建议可以把下拉給搜索。
虽然仁者见仁智者见智，我还是想说，底部tab的形式不适合印象笔记。 🔗

印象笔记，你说！你为什么要这么做？
我才被wiz恶心到，从wiz搬过来，你现在却变得跟他一个样子，你想提醒我啥？ 🔗

没想到都到 2017 年了，居然还有那么多人质疑 Tab Bar 的合理性。虽然我早已不用 Evernote，也不想对它这次更新发表什么看法，但有一点毫无疑问：如果 Evernote 这次更新有什么问题，那也应该是其他方面的问题，而不会是 Tab Bar，因为它是一个好设计，尤其是在 iOS 上。

虽然我们应该鼓励个性，追求独特与品味，虽然 App Store 并不乏交互十分新颖且高效的新 app，但不得不说，对于拥有大量用户的人气 app，只有从长远考虑，选择更加普世、更加合理的「最佳实践」，才能让 app 稳定发展，以满足更广大用户的需求。

毫无疑问，Tab Bar 就是移动应用 UI 设计最佳实践的一部分。

不用 TabBar 的设计效率真的不高啊。在 OmniFocus for iOS 某个目录的某个项目的某个任务当中，我想快速跳到首页，必须：后退 -> 后退 -> 后退 -> 后退…真是抓狂。
